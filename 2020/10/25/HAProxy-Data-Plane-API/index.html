HAProxy Data Plane API

在使用 HAProxy 時，常常需要針對不同的服務進行不一樣的配置，如修改 backend server 或 load balancing mode 等。因此 HAProxy 針對動態配置提供一套 API 介面，讓使用者可以透過 RESTful API 方式動態更改 HAProxy 的配置，更詳細的說明可以參考 haproxytech/dataplaneapi。

環境

OS Host Name Private Network
CentOS 7 HAProxy 192.168.5.2
CentOS 7 httpd1 192.168.5.3
CentOS 7 httpd2 192.168.5.4

安裝 HAProxy

下載 HAProxy 2.2.4(LTS) 並解壓縮

進入 haproxy-2.2.4 目錄並針對主機作業系統進行編譯

安裝需要 make 與 gcc 兩個套件，若是使用 CentOS 的作業系統可利用 yum install 指令進行安裝。

安裝 HAProxy 2.2.4

建立 HAProxy 紀錄的靜態目錄

建立 symbolic link

將 HAProxy 交由 systemd 進行管理

建立 haproxy use 與 group

建立 /etc/haproxy/haproxy.cfg 檔案並寫入基本配置

啟動 HAProxy 服務

到這裡，基本的 HAProxy 已經安裝完成，接下來進行 HAProxy Data Plane API 安裝

安裝 HAProxy Data Plane API

下載 dataplaneapi

將 build/dataplaneapi 檔案複製至 /usr/local/bin 目錄

在 /etc/haproxy/haproxy.cfg 的 global 中加入 master-worker ，並在檔案最底部加入 userlist 與 dataplaneapi 配置

修改完成後，重新啟動 haproxy

啟動後可透過 http://127.0.0.1:5555/v2/docs 參閱可用的 APIs
